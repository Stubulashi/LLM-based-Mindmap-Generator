# 课程思维导图生成质量评估标准（Evaluation Criteria for Lecture Mind Map Generation）

## 1. 基于“真实大纲”的结构对齐（Alignment with Ground Truth）

这是最直接、最客观的评估维度。在一节正规的Lecture中，教授通常会提供PPT或课程大纲（Syllabus），其中的标题与副标题天然构成了一张“完美树状图”（Gold Standard）。评估逻辑是：将Agent生成的节点结构与教授官方PPT的目录结构进行比对，衡量二者的重合程度。
This is the most direct and objective evaluation dimension. In a standard lecture, the instructor almost always provides slides or a syllabus, whose titles and subtitles naturally form a “gold‑standard” tree. The evaluation logic is straightforward: compare the node structure generated by the agent with the official outline and measure the degree of overlap.

具体指标：
Specific metrics:

- **层级准确率（Hierarchical Accuracy）**：Agent是否正确识别了主要概念（一级节点）与补充细节（二级、三级节点）之间的从属关系？即生成导图的层级逻辑是否与官方大纲保持一致。  
  *Does the agent correctly identify the parent‑child relationships between main concepts (first‑level nodes) and supporting details (second‑ and third‑level nodes)? In other words, does the hierarchy of the generated mind map align with the official outline?*

- **树编辑距离（Tree Edit Distance）**：计算将Agent生成的导图转换为“官方PPT大纲树”所需的最少编辑操作（增加、删除、修改节点）次数。距离越小，说明Agent提取的结构越贴近教授真实的授课逻辑。  
  *Compute the minimum number of edit operations (insertions, deletions, substitutions of nodes) required to transform the agent‑generated mind map into the “official PPT outline tree”. A smaller distance indicates that the extracted structure more closely follows the instructor’s actual lecture flow.*

## 2. 核心学术概念的“召回率”（Concept Recall & Information Extraction）

Lecture的根本目的是传授知识点。无论生成的导图形式多么精美，若遗漏了关键定义，即为失败。评估逻辑：从该节Lecture中预先定义10–20个必须掌握的核心概念（例如本课程中的LLM、Agent、MCP、ReAct、Sub‑graph Retrieval等），逐项检查它们是否作为关键节点出现在导图中。
The primary goal of a lecture is knowledge delivery. No matter how visually appealing a mind map may be, it fails if critical definitions are omitted. The evaluation logic is as follows: pre‑identify 10–20 essential concepts that must be mastered from the lecture (e.g., LLM, Agent, MCP, ReAct, Sub‑graph Retrieval in this course), and then check whether each of them appears as a key node in the generated mind map.

具体指标：
Specific metric:

- **实体召回率（Entity Recall）** = 导图包含的核心概念数量 ÷ 教授实际讲授的核心概念总数。该指标直接衡量模型在长文本处理中是否避免了关键信息的“遗忘”。  
  *(Number of essential concepts present in the mind map) / (Total number of essential concepts delivered by the instructor). This metric strongly demonstrates the model’s ability to retain crucial information over long contexts without “forgetting”.*

## 3. 下游任务测试：开卷问答效能（Downstream Task: QA Utility）

在计算语言学中，评估一种数据结构优劣的最有效方法之一，是考察它在“下游任务”中的表现。课程思维导图的最终目的是辅助学生复习与快速查阅，因此我们不直接评价导图本身，而是评价“基于该导图回答问题的能力”。
In computational linguistics, one of the most effective ways to evaluate a data structure is to examine its performance on a downstream task. The ultimate purpose of a course mind map is to assist with review and quick reference; therefore, rather than judging the mind map itself, we assess “the ability to answer questions based on this mind map.”

**具体操作（LLM模拟考试）**  
**Procedure (LLM‑simulated exam)**

- 准备10道关于本节Lecture的测验题。  
  *Prepare 10 quiz questions about the lecture.*
- **对照组**：让一个基础LLM阅读数万字的原始逐字稿来回答这10道题，记录答题准确率以及所消耗的Token／计算时间。  
  *Control group: Provide a base LLM with the full raw transcript (tens of thousands of words) and have it answer the 10 questions. Record its accuracy and the associated token consumption / computation time.*
- **实验组**：让LLM仅阅读生成的导图JSON数据来回答同样的10道题。  
  *Experimental group: Provide the same LLM with only the generated mind‑map JSON data and have it answer the same 10 questions.*

**预期数据与论文价值**  
**Expected outcome and paper impact**

若能证明“仅依靠导图的实验组”不仅答题准确率与对照组相当（甚至因去除了冗余信息而更高），同时API调用成本降低约80%，这便构成一项极具说服力的评估结果，完全能够作为论文的核心亮点。
If you can demonstrate that the experimental group—using only the mind map—achieves accuracy comparable to (or even higher than, due to the removal of noise) that of the control group, while reducing API call costs by roughly 80%, this becomes an exceptionally strong evaluation result and can directly serve as a core highlight of a paper.

## 4. 生成效率与语音转录保真度（Generation Efficiency & Transcription Fidelity）

除导图本身的质量外，系统的实用价值同样取决于处理速度与上游语音识别（STT）的可靠性。因此，我们将端到端生成延迟以及语音转录信息的完整性一并纳入评估体系。
Beyond the quality of the mind map itself, the practical value of the system also hinges on processing speed and the reliability of upstream speech‑to‑text (STT). We therefore incorporate end‑to‑end generation latency and transcription completeness into the evaluation framework.

评估逻辑：从用户上传音频文件开始计时，直至思维导图生成完毕，记录全流程耗时；同时，检查STT转换环节是否因识别错误或遗漏而导致关键知识点丢失，从而制约下游导图质量的上限。
Evaluation logic: Measure the total time from the moment an audio file is uploaded until the mind map is fully generated. In parallel, examine whether the STT stage introduces critical knowledge loss due to recognition errors or omissions, thereby capping the downstream mind‑map quality.

具体指标：
Specific metrics:

- **端到端生成时间（End-to-End Generation Time）**：从音频文件上传完成到最终思维导图输出所经历的总时长，反映系统整体的响应效率。  
  *The total elapsed time from audio file upload to final mind map output, reflecting the overall responsiveness of the system.*
- **语音转录内容保存率（Transcription Content Retention Rate）**：预先标定音频中必须准确转写的关键概念集合（如术语、定义、公式等），计算STT输出文本实际保留这些概念的比例，即 `保存率 = 成功转写的关键概念数 ÷ 应转写的关键概念总数`。该指标衡量上游语音识别对知识完整性的保障程度。  
  *Pre‑define a set of key concepts (terms, definitions, formulas, etc.) that must be accurately transcribed from the audio. Then compute the proportion of these concepts actually retained in the STT output — i.e., `Retention Rate = (Number of successfully transcribed key concepts) / (Total number of key concepts that should be transcribed)`. This metric gauges how well the upstream STT preserves knowledge integrity.*
